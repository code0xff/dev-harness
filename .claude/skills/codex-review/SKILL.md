# Codex Review

구현 완료 후 OpenAI Codex CLI로 변경 사항을 리뷰하고, 결과를 평가하여 타당한 지적은 코드에 반영한다.
반복 실행이 필요하면 `/ralph /codex-review`로 ralph에 위임한다.

## 0. 사전 확인

- `codex` CLI가 설치되어 있는지 확인한다 (`which codex` 또는 `codex --version`).
- 설치되어 있지 않으면 사용자에게 알리고, `/self-review`로 대체할지 확인한다.
- codex 명령 실행이 실패하면 (네트워크 에러, 인증 실패 등) 에러 내용을 사용자에게 보고하고 진행 여부를 확인한다.

## 1. Codex 리뷰 실행

unpushed commit을 대상으로 `codex review --base <upstream>` 명령을 실행한다.

- unpushed commit이 없으면 "리뷰 대상 없음"을 보고하고 즉시 종료한다.
- `git fetch`로 remote 상태를 최신화한다. 실패하면 경고를 출력하고 기존 local ref로 진행한다.
- remote tracking branch를 기준점으로 결정한다.
  - `git rev-parse --abbrev-ref @{upstream}`로 tracking branch를 확인한다.
  - tracking branch가 없으면 `git remote show origin | grep 'HEAD branch'`로 remote 기본 브랜치를 조회하고, `origin/<branch>` 형태로 사용한다.
  - 위 방법이 모두 실패하면 사용자에게 base branch를 직접 지정하도록 요청한다.
- 인자가 전달되면 (`/codex-review <base-branch>`) `codex review --base $ARGUMENTS`로 실행한다.
- 인자가 없으면 `codex review --base <upstream>`으로 실행한다.

## 2. 리뷰 결과 평가

Codex가 출력한 리뷰 결과를 읽고, 각 지적 항목을 다음 기준으로 분류한다:

- **즉시 반영**: 버그, regression, 명확한 설계 위반, 누락된 에러 처리 등 객관적으로 문제가 있는 항목. 코드에 직접 수정을 적용한다.
- **사용자 판단 필요**: 트레이드오프가 있거나, 의도적 선택일 수 있거나, 현재 작업 scope과 맞지 않는 항목. 수정하지 않고 보고만 한다.
- **무시**: 스타일 선호, 범위 밖 개선 제안, 근거가 약한 지적. 개별 내용은 보고하지 않되, 건수와 대표 사유는 보고 형식에 포함한다.

평가 시 다음을 우선순위 순으로 참고한다 (충돌 시 상위가 우선):

1. 프로젝트의 architecture 문서 (있는 경우)
2. roadmap / workstream 문서 (있는 경우)
3. `.claude/rules/` — 현재 구현 규칙
4. 현재 작업의 scope 문서 (있는 경우)

## 3. 수정 적용

- uncommitted 변경이 있으면 리뷰 수정분만 staging하여 기존 작업이 섞이지 않게 한다.
- **즉시 반영** 항목은 코드를 직접 수정한다.
- 수정 후 프로젝트의 빌드 검증과 테스트를 실행하여 수정이 다른 것을 깨뜨리지 않았는지 확인한다.
- 테스트가 실패하면 원인을 먼저 분류한다:
  - 수정이 원인인 경우: 수정을 되돌리고 **사용자 판단 필요**로 재분류한다.
  - 기존 flaky 테스트나 환경 문제인 경우: 수정을 유지하고 실패 원인을 보고한다.
- 수정이 완료되면 수정 사항을 commit한다.

## 4. 중단 기준

다음 중 하나라도 해당하면 push 진행을 중단하고 사용자에게 보고한다:

- 심각도 높은 버그가 발견되고 즉시 반영으로 해결되지 않은 경우 (보안, 데이터 손실, 인증/권한 우회)
- 동일 변경에서 버그가 2개 이상 발견되고 즉시 반영으로 해결되지 않은 경우
- 설계 원칙 위반이 발견된 경우
- 수정 적용 후 빌드 또는 테스트가 실패하고, 단순 수정으로 해결되지 않는 경우
- 동일한 지적이 수렴하지 않고 계속 발생하는 경우 (순환 판단)

## 5. 보고 형식

리뷰 결과를 사용자에게 다음 형식으로 보고한다:

### Codex 리뷰 원문 요약
(Codex가 지적한 항목 수와 주요 내용 요약)

### 즉시 반영한 항목
(각 항목: 파일, 변경 내용, Codex 지적 원문, 반영 이유. 없으면 `없음`)

### 사용자 판단이 필요한 항목
(각 항목: 파일, Codex 지적 원문, 판단을 미룬 이유. 없으면 `없음`)

### 무시한 항목
(건수와 대표 사유. 없으면 `없음`)

### 리뷰 요약
(전체 변경 품질에 대한 1-2문장 평가)
